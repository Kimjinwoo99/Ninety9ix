# 보안 가이드

## GitHub에 업로드하기 전 확인사항

### ✅ 반드시 제거해야 할 항목

1. **API 키 및 인증 정보**
   - OpenAI API 키
   - 기타 서비스 API 키
   - 데이터베이스 비밀번호
   - JWT 시크릿 키

2. **개인 정보**
   - 실제 사용자 데이터
   - 개인 식별 정보 (주민번호, 전화번호 등)
   - 이메일 주소

3. **하드코딩된 민감한 설정**
   - 프로덕션 데이터베이스 URL
   - 내부 서버 주소
   - 관리자 계정 정보

### 🔒 환경 변수 사용

모든 민감한 정보는 환경 변수나 `.env` 파일을 통해 관리하세요.

1. `.env.example` 파일을 참고하여 `.env` 파일 생성
2. `.env` 파일은 `.gitignore`에 포함되어 있어 Git에 업로드되지 않습니다
3. 실제 API 키는 `.env` 파일에만 저장

### 📝 체크리스트

업로드 전 다음을 확인하세요:

- [ ] 모든 API 키가 코드에서 제거되었는가?
- [ ] `.env` 파일이 `.gitignore`에 포함되어 있는가?
- [ ] `.env.example` 파일이 제공되어 있는가?
- [ ] 하드코딩된 비밀번호나 토큰이 없는가?
- [ ] 실제 사용자 데이터가 샘플 데이터로 대체되었는가?
- [ ] 로그 파일에 민감한 정보가 포함되지 않았는가?

### 🚨 만약 실수로 민감한 정보를 업로드했다면

1. 즉시 해당 정보를 변경하세요 (API 키 재발급 등)
2. Git 히스토리에서 제거:
   ```bash
   git filter-branch --force --index-filter \
   "git rm --cached --ignore-unmatch 파일경로" \
   --prune-empty --tag-name-filter cat -- --all
   ```
3. GitHub에서 해당 커밋을 삭제하고 force push
